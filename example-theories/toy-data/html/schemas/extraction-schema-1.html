Extraction Schema Details for extraction-schema-1

Extraction Schema (General Information)

Schema ID: extraction-schema-1

Extraction Query: Extract any mentions of language model agents augmented with memory for playing text games, including details of memory type, integration, and performance comparisons.

Extraction Schema (Details)

Field Name Type Description
agent_name str The name of the language model agent evaluated in the paper (e.g. 'KG-A2C', 'LLM-Agent', etc.).
agent_description str A brief description of the agent, including its architecture and purpose.
model_name str The name of the underlying language model used by the agent (e.g. 'GPT-3', 'T5', etc.), if specified.
memory_type str The type of memory augmentation used (e.g. 'episodic memory', 'external memory', 'retrieval-augmented', 'context window extension', etc.).
memory_integration str How the memory is integrated into the agent (e.g. 'retrieval from external database', 'recurrent state', 'context window concatenation', etc.).
benchmark_name str The name of the text game or benchmark used to evaluate the agent (e.g. 'TextWorld', 'Jericho', etc.).
performance_with_memory str The performance of the agent on the benchmark when using memory augmentation (include metric and units, e.g. 'score: 85/100', or 'success rate: 70%').
performance_without_memory str The performance of the agent on the benchmark without memory augmentation (include metric and units, or null if not reported).
has_performance_comparison bool Does the paper report a direct comparison of performance with and without memory augmentation? (true, false, or null if not specified)
key_findings str A concise summary of the key findings regarding memory augmentation for LLM agents in text games.
challenges_or_limitations str Any reported challenges, limitations, or negative results related to memory augmentation in this context.
