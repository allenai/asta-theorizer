Theory (General Information)

ID: theory-6

Name: Compression and Selection Mechanisms Are Essential to Overcome Scaling Limits for In-Context Memory in LLM Text-Game Agents

Type: general

Theory Query: Build a theory of how language model agents can best be augmented with memory to perform well at text games, based on the following results.

Description: This theory posits that as text-game environments scale in observation length, object count, and task horizon, LLM and RL-based agents must rely on memory compression (e.g., diff-history, summarization, area selection) and relevance-based selection (e.g., retrieval, recency/importance scoring) rather than full retention of episode histories in prompt context. Without these, agents hit hard resource and sample efficiency ceilings due to context-window length, token costs, and cognitive overload. Compression must not discard critical state changes; selection must ensure salient, task-relevant cues are retained.

Theory (Statement/Laws)

Theory Statements/Laws

Statement 0: Compression and Relevance Law for Scalable LLM Memory

Statement: When environment text or underlying state grows faster than available LLM/RL context capacity, agents employing memory compression (e.g., diff-history, summarization, area-pruning) and relevance-based selection (e.g., scoring by recency, importance, embedding similarity) maintain or improve performance compared to full-history or fixed-window approaches, and do so with lower compute, lower error rate, and greater sample efficiency.

Domain/Scope: Applies to LLM-based and RL+LM agents in high-dimensional, long-horizon, or multi-step domains where prompt token or recurrence-hidden-state memory is limiting, including NetHack (LangHack), TextWorld, Goedendag, and real-world multi-step/embodied text settings.

Special Cases

  1. Low-dimensional or toy games may not require compression.
  2. Compression methods must preserve critical information; lossy/over-aggressive compression can degrade reasoning.

Supporting Evidence for this Law

  1. Diff-history agents greatly outperform full-history on NetHack (e18.0, e18.1, e18.2); area-selection and summarization improve Goedendag win-rate and error reduction (e12.0, e12.1, e12.2). Full-history and in-context-only agents perform poorly on long/hard TextWorld variants and rapidly hit prompt limits (e5.2). Score-based selection (Simulacra, MemInsight), attribute-based filtering and embedding retrieval (e23.0), and pruning of action/state space (KG-A2C masking, e11.3) consistently reduce errors and improve efficiency. Theory (Additional Details)

New Predictions (Likely outcome)

  1. Any agent using full-history or sliding-window memory on a new, deep text-based game (e.g., complex NetHack, long-horizon RPG) will rapidly saturate context and falter, while diff/area/summarization-based agents maintain high task completion rates.
  2. Combining relevance scoring (e.g., recency+importance retrieval) with compressed memory will further accelerate learning and reduce errors in knowledge-intensive tasks and generation-centric games.
  3. Structured selection (embedding or attribute-based) will outperform random or uniform sampling in multi-agent memory fusion tasks.

New Predictions (Unknown outcome/high-entropy)

  1. Advanced lossy compression (e.g., learned/discrete representations or language-model-based summarization) may achieve a favorable compute/performance tradeoff up to hundreds of thousands of steps—potentially enabling LLM agents to span whole-game or cross-episode contexts.
  2. Introducing multimodal (image, table, audio) summarization together with token-level compression in complex simulation games will outperform text-only compressions.
  3. Combining compression and retrieval in a hierarchical memory pipeline will enable emergent meta-cognitive behaviors like temporal abstraction or automatic subgoal discovery.

Negative Experiments

  1. If a full-context or naive summarization agent outperforms compressed (diff-based/area-based) or relevance-selected memory in large, high-branching, partially observable text games.
  2. If over-compression or poorly selected memories decrease agent performance below a memoryless baseline.
  3. If relevance/score/embedding-based retrieval fails to improve QA, search, or planning tasks compared to uniform sampling or local windowing across diverse text games.

Unaccounted for Evidence
